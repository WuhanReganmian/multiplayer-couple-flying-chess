# 🚀 多人情侣飞行棋 需求文档

## 1. 文档概览
* **产品定位：** 一款用于情侣或多对情侣之间增强亲密度、引导情趣互动的数字化飞行棋游戏。
* **核心目标：** 通过随机的游戏机制，引导玩家从轻微互动逐步升级到深度亲密行为。
* **核心限制：** 仅限 **异性** 之间互动，严禁同性互动。

## 2. 目标用户与场景
* **用户场景：** 卧室、度假酒店等私密空间。
* **玩家规模：** 2 - 4 人。
* **匹配逻辑：**
* **2人模式：** 1男 + 1女（固定互动对象）。
* **3-4人模式：** n男 + n女（随机匹配异性对象，至少一名异性）。

## 3. 功能需求

### 3.1 角色创建与匹配 (Onboarding)
* **性别选择：** 玩家进入游戏必须选择【男】或【女】。
* **人数校验：**
* 总人数至少为 2 ，且必须有 1男1女，否则无法开始。
* **互动对象池：**
* 当需要触发任务对象时，从该池中随机抽取一个与当前玩家性别相反的 ID。

### 3.2 游戏核心机制 (Core Gameplay)
* **棋盘结构：** 环形路径，包含若干个随机“任务格”，最先到达终点的人为获胜。
* **移动逻辑：** 随机掷骰子 -> 移动对应步数 -> 触发当前格子的任务(可选择开始任务或拒绝任务，如果拒绝则回退1-3步，重新触发)。
* **任务分级 (Task Tiering)：**
任务根据棋盘进度分为 **5 个阶段**，每名玩家都可以单独设置阶段：

| 阶段 | 分类名称 | 互动强度 | 任务示例 (仅供参考) |
| :--- | :--- | :--- | :--- |
| **L1** | **破冰** | $\text{Low}$ | 互相对视 30 秒，分享一个对方的优点。 |
| **L2** | **暧昧挑逗** | $\text{Mid-Low}$ | 在对方耳边低语，轻触对方颈部。 |
| **L3** | **褪衣相亲** | $\text{Mid}$ | 对方决定你脱掉一件衣物，或进行 1 分钟肌肤接触。 |
| **L4** | **边缘取悦** | $\text{Mid-High}$ | 进行非插入式的深度挑逗，直至对方发出快感信号。 |
| **L5** | **实战冲刺** | $\text{High}$ | 进入正式性行为阶段，由任务指定体位或节奏或使用情趣道具或执行特定的 Roleplay 指令。 |

每一个阶段预设100个不同的任务（共计500条），并且可以在设置界面进行 新增、编辑、删除 操作

### 3.3 多人随机互动逻辑 (Multiplayer Logic)
* **触发指令：** 当格子任务为 `[随机对象] + [任务内容]` 时：
* **逻辑流：** `当前玩家` -> `检索异性池` -> `随机抽取一名异性` -> `指派为本次互动对象`。
* **结果输出：** 界面提示：“【玩家 A】请对 【玩家 B】执行 [任务 X]”。

## 4. 交互设计 (UI/UX)
* **氛围营造：** 采用暗色调（深紫、暗红、黑色），营造私密诱惑感。
* **任务卡片：** 任务以“翻牌”形式出现，增加未知带来的刺激感。
* **进度指示：** 棋盘背景色随阶段 (L1  L5) 逐渐由浅色变为深红色，暗示强度增加。

## 5. 非功能性需求
* **私密性：** 本地运行，不上传任何玩家互动数据至服务器。
* **响应速度：** 掷骰子与任务随机生成的动画需流畅，无卡顿。
* **自定义能力：** 支持管理员（房主）在开始前，对每个分类的任务库进行自定义修改或删除。
